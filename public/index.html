SQL Analytics, All In One Place

A database analytic tool that compares the efficiency of different schemas and queries on a granular level to make better informed architectural decisions regarding SQL databases at various scales.

The SeeQR interface.
CREATE TABLE public.address (
"_id" serial NOT NULL,
"zipcode" integer NOT NULL,
"street" varchar NOT NULL,
"city" varchar NOT NULL,
"state" varchar NOT NULL,
CONSTRAINT "address_pk" PRIMARY KEY ("_id")
) WITH (
OIDS=FALSE
);

Custom Schema Generation

Compare efficieny of schemas by uploading pre-existing .tar or .sql files, or input script directly into application. Database and schemas are generated in a 1:1 relation to maintain scalability, designed for testing.

Data Visualization

Data visualization for query-based analytics are available within the context of individual schemas, and across any schemas uploaded into application for cross-schema comparisons. Comparison tables are easily customizable through custom query label look-up.

Containerization

SeeQR streamlines the process of instantiating postgres databases by leveraging Docker containerization. This means instances of databases are automatically created every time new schema data is uploaded or inputted via the SeeQR GUI. At the click of a button.
